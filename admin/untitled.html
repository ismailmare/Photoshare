Installation

1. Download project.tgz and put it into folder you want your web document (web_docs) to be created in .

2. Extract the file in the folder using command: tar xzvf project.tgz

3. Set Permissions of contents of web_docs using command: chmod -R 777 web_docs (NOTE: Not secure with 777 permissions can adjustment if choose)

4. Go into the web_docs folder using command: cd web_docs

5. Install database using makefile with command: make install u=”oracleusername” p=”oraclepassword”

User Manual

Main Page (Index.html)

This is the main page (index.html) for the website. There are two options to go login with your username and password or go to the user documentation. You can go to these option by clicking the buttons labeled User Login and User Documentation.

Login Module

Once you have navigated to the User Login you will be presented with a form to input your username and password. Once you have entered your username and password click the login button. If your username and password are correct it should log you in to your login information.

Login Information (Login Module)

Depending on your class you will be presented different options for buttons which lead to all of the other modules on the radiology website:

Logout (Login Module)

This button is present to all classes on the top of the login information page. Clicking this button will simply logout the user.

Modify Account (Login Module)

This button is present to all classes on the top of the login information page under the logout button. This button navigates the user to another page where a user can modify their account.

This option is used to update the user email they are using. They can do this by filling in the form (shown above) with the email they wish to change to and click Update Email.

Change Password

This button is present to all classes on the top of the login information page under the logout button. This button navigates the user to another page where a user can change their password.

This option is used to change the password they are using. They can do this by filling in the password they wish to change to in form (shown above) and clicking Update Email.

User Management

Insert Person (User Management Module)

This button is present to only users with the class administrator present. This button navigates the administrator to another page where they can add a person to the database.

On this page there is a form where an administrator has to entering inputs first name, last

name, email, address, and phone. Once the administrator has successfully done this they can click Create person. This person will then be added to the database.

Update Person (User Management Module)

This button is present to only users with the class administrator present. This button navigates the administrator to another page where they can update a person in the database.

In this form it first asks you to enter a email to the person you want to update and press Find User. Once the person is successfully found it will open another form that is populated with the current person data. You can edit this information as you choose and then click Update User to update the person.

Insert User (User Management Module)

This button is present to only users with the class administrator present. This button navigates the administrator to another page where they can add a user to the database.

On this page there is a form where an administrator has to entering inputs a persons email, username, password, and a class they will be under. Once the administrator is finished entering information they can click create user and they will be added to the database.

Update User (User Management Module)

This button is present to only users with the class administrator present. This button navigates the administrator to another page where they can update a user in the database.

In this form it first asks you to enter a username to the user you want to update and press Find User. Once the user is successfully found it will open another form that is populated with the current user data. You can edit this information as you choose and then click Update User to update the user.

Insert Family Doctor (User Management)

This button is present to only users with the class administrator present. This button navigates the administrator to another page where they can add a family doctor to the database.

On this page there is a form where an administrator has to entering inputs the email of the patient and doctor they want to add. Once the administrator is finished entering information they can click “Create family doctor” and they will be added to the database.

Update Family Doctor (User Management Module)

This button is present to only users with the class administrator present. This button navigates the administrator to another page where they can update a family entry in the database.

On this page there is a form where an administrator has to entering inputs the email of the patient and doctor they want to update and click “Search family doctor. Once it has searched successfully it will open up another form asking the administrator to enter a patient email and doctor email they want to update the family doctor entry with.

Generate Report (Report Generating Module)

This button is present to only users with the class administrator present. This button navigates the administrator to another page where they can generate a report based on a diagnostic.

On this page there is a form where an administrator has to entering inputs the diagnostic of report they want to generate and click “Find Reports”. Once it has searched successfully it will generate a report of name, address, phone, and test date.

Insert Radiology Record (Uploading Module)

This button is present to only users with the class radiologist present. This button navigates the administrator to another page where they can insert a radiologist record.

On this page there is a form where a radiologist has to enter the inputs email of doctor, email of patient, test type, test date, prescribing data, diagnostic, description, and any pictures the radiologist chooses to upload. To upload image click the browse button and a file manager should open. Within that you can select an image or multiple images holding (CTRL) and select what images you want. Once the radiologist has uploaded what she wants she can click “Create radiology record” and the record will be created.

Search Radiology Record (Search Module)

This button is present to all users. This button navigates the user to another page where they can search for radiology records

On this page there is a form where users can input keywords they want to be searched within radiology records. Also, a user can choose time periods to search for radiology records in between along with listing the records by Most Recent First/Most Recent Last. Once the user has parameter of search they want they can click “Search” and it will create a list based upon results of your search.

Data Analysis (Data Analysis Module)

This button is present to only users with the class administrator present. This button navigates the administrator to another page where they can do data analysis.

On this page there is a form where users can input whether or not they want to count the number of images a certain groups have Patient Id, Test Type, and Time (including weekly,monthly, and yearly). Once the administrator has selected what groups they want the data analysis on they can click “Generate Report and a table will be generated based upon the groups selected.
